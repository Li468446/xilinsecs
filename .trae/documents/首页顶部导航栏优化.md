# 首页顶部导航栏优化计划

## 问题描述
产品中心下拉的二级菜单栏（安全产品、安全解决方案、安全服务、其他产品）不可点击，只有三级导航栏才可以点击。

## 问题分析

当前导航栏结构：
```
产品中心
  └── 安全产品 (有链接 /business/product/security/) ← 二级，不可点击
        └── 数据跨境合规审查 ← 三级，可点击
        └── 信创防火墙 ← 三级，可点击
        └── 灵契授权中心 ← 三级，可点击
  └── 安全解决方案 (有链接 /business/product/solutions/) ← 二级，不可点击
  └── 安全服务 (有链接 /business/product/services/) ← 二级，不可点击
  └── 其他产品 (有链接 /business/product/other/) ← 二级，不可点击
```

**问题原因：**
- 二级菜单项有 `href` 链接，但整个 `<li>` 元素hover时会显示三级菜单
- 由于hover区域覆盖整个 `<li>`，用户无法触发链接的hover效果
- 点击时会同时触发hover显示三级菜单，导致无法跳转到二级菜单的链接页面

## 解决方案

修改 `index.html` 中的产品中心导航结构：
1. 二级菜单项（安全产品、安全解决方案、安全服务、其他产品）**去除链接**
2. 将链接转移到专门的可点击文字上
3. 保持原有的hover显示三级菜单功能

## 实施步骤

1. 修改 `index.html` 第134-176行
   - 移除二级菜单项 `<a>` 标签的 `href` 属性
   - 二级菜单项保持hover显示三级菜单的功能
   - 三级菜单项保持可点击跳转功能

2. 调整CSS样式（第321-335行）
   - 确保 `.submenu-link` 在二级菜单项中可点击
   - 添加适当的cursor样式区分可点击和不可点击状态
